Как выполнять задание
Вам необходимо создать техническое задание на разработку интернет-магазина для завода лампочек (20 наименований) с возможностью добавлять товары в корзину, оформлять заказы и управлять товарами через панель управления (админку).

ТЗ должно включать:

Прототипы всех необходимых страниц в Figma или аналогичном редакторе.
Документацию:
к микросервису управления товарами;
микросервису управления заказами;
микросервису панели управления.
У каждого микросервиса должно быть два вида документации:

описание структуры базы данных;
документация к API в формате Postman.
Важно. В домашних заданиях 2–4 вы должны будете реализовать написанное техническое задание. Заложите в него те функции, реализацию которых хотите отработать.

Формат сдачи и отправка задания
У вас должна быть подготовлена папка со ссылками на все артефакты: прототипы веб-страниц, описание структуры БД и документацию к API каждого микросервиса. Обязательно добавьте текстовый файл с фамилиями всех исполнителей. Не забудьте открыть для папки доступ на комментирование.

Оценивание:
Критерии | Описание | Баллы
Ключевые страницы (каталог, карточка, корзина, заказ, подтверждение)

Отсутствует большинство страниц — 0
Есть часть страниц, но не все или страницы не связаны — 1
Есть все страницы, логика полностью покрыта — 2

Интерфейсы управления товарами и заказами

Отсутствуют — 0
Реализованы частично — 1

User flow и навигация

Переходы нелогичные или отсутствуют — 0
Есть навигация, но с проблемами — 1

Структура БД (каталог)

Не соответствует прототипам — 0
Частично покрывает данные — 1
Полностью соответствует прототипам — 2

Модель заказов (заказы, позиции, статусы)

Не описана или некорректна — 0
Частично описана — 1
Полная и корректная модель — 2

Основные действия (CRUD, корзина, заказ)

Отсутствуют ключевые операции — 0
